# Retail Technology Show Exhibitors Scraper

A robust, clean data scraper for extracting exhibitor information from JSON-LD structured data.

## Quick Start

```bash
pip install -r requirements.txt
python scraper.py
```

## Usage Examples

### Scrape a single page
```python
scraper = ExhibitorScraper("https://www.retailtechnologyshow.com/exhibitors")
scraper.run(start_page=13, end_page=13)
```

### Scrape all pages (1-17)
```python
scraper.run(start_page=1, end_page=17)
```

### Scrape with custom output file
```python
scraper = ExhibitorScraper(
    base_url="https://www.retailtechnologyshow.com/exhibitors",
    output_file="my_exhibitors.csv"
)
scraper.run(start_page=1, end_page=5)
```

## Output

CSV file with two columns:
- **name**: Company name (string)
- **url**: Company profile or website URL (string)

### Example Output
```
name,url
RGIS Inventory Specialists,https://www.rgis.co.uk
Ricoh UK,https://www.ricoh.co.uk/
Roke,https://www.retailtechnologyshow.com/exhibitors/roke
```

## How It Works: Inspect Element & Code

### What is Inspect Element?

When you **right-click → Inspect Element**, you're viewing the live **HTML DOM** (Document Object Model) that the browser has rendered. This is what the website's code sent, plus any JavaScript modifications.

### How Code Accesses It

```
Raw HTML from Server
    ↓
Browser Parses HTML + Runs JavaScript
    ↓
DOM Tree Created (what you see in Inspector)
    ↓
Your Code Fetches HTML & Parses It
```

### Three Ways to Access Data

| Method | How It Works | When to Use |
|--------|-------------|------------|
| **Static HTML Parsing** | Fetch HTML with `requests`, parse with BeautifulSoup | Data is in raw HTML (like our JSON-LD) ✓ FASTEST |
| **Headless Browser** | Use Selenium/Playwright to render JavaScript | Page content generated by JavaScript after load |
| **API Direct** | Examine Network tab, call the API directly | Website uses AJAX to fetch data |

### For This Project: JSON-LD in HTML

The exhibitor data is embedded as **static JSON-LD** in `<script type="application/ld+json">` tags in the raw HTML:

```html
<script type="application/ld+json">
{
  "@type":"ProfilePage",
  "mainEntity":{
    "@type":"Organization",
    "name":"RGIS Inventory Specialists",
    "url":"https://www.rgis.co.uk",
    "logo":"...",
    "sameAs":["...social links..."]
  }
}
</script>
```

**Our scraper:**
1. Fetches the raw HTML
2. Finds all `<script type="application/ld+json">` tags
3. Parses the JSON directly
4. Extracts `name` and `url`
5. Validates data (all-or-nothing approach)
6. Saves to CSV

No browser rendering needed = **Fast & reliable**.

## Data Quality (All or Nothing)

The scraper only saves exhibitors where:
- ✓ Company name exists AND
- ✓ Company URL exists

If either is missing, the entry is skipped (not saved with null values).

## Adaptability

To modify for a different website or schema:

1. **Identify the data structure** (Inspect Element → look for `<script type="application/ld+json">`)
2. **Update `BASE_URL`** in the scraper
3. **Adjust field extraction** in `_extract_from_profile_page()` or `_extract_from_item()` if schema differs
4. **Modify CSV fieldnames** in `save_to_csv()` if you need different columns

## Troubleshooting

**"No companies found on page X"**
- The page may not have JSON-LD data, or the structure differs
- Check Inspect Element on the website to verify the schema

**"HTTP Error 403/429"**
- Add delays between requests or reduce concurrent requests
- Check if the site blocks automated access

**Duplicate entries**
- Automatically removed by URL deduplication before saving
